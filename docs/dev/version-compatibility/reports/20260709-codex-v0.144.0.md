## Execution phase

### Phase 1: Enhance source code change detection

1. Extend the extractor to capture the complete web-search tool field, `AdditionalTools` wire shape, `ModelMessages.approvals` and the key capability values of the bundled model.
2. Reserve independent fields for tag commit, synchronized source commit and package release version.
3. Add extractor fixture to ensure that field rename, nested schema and catalog value changes enter "changed".

### Phase 2: Fix blocking compatibility points

1. First implement Responses Lite request parsing: recovery tools, developer instructions, history and reasoning context; define deduplication and priority when top-level/embedded tools are mixed.
2. Make `remove_image_generation` recognize both the old `image_generation` and the new `image_gen` namespace/function, and handle `additional_tools` inline definitions.
3. Update the new web-search field and the two paths of hosted/extension; fix the `bio_policy` error contract.
4. Add `ultra` reasoning and provider-specific downgrade; keep the encrypted/summary status for continuous rounds.

### Phase 3: Tightening history, deferred tools and streams

1. Add deterministic regression to the bridge's item ID, call ID, added/done/completed associations.
2. Cover the state isolation of canonical/extension TurnItem, MCP snapshot, image namespace, compact/resume/ fork/restart and two windows.
3. Only when the header life cycle changes are actually captured, the window key is extended from header to body `client_metadata`; state ownership is not changed based on guessing.

### Phase 4: Run All Automated Gates

1. Run the Codex special pytest, including the new Lite/catalog/web-search/error/history fixtures added in this report.
2. Run `make lint` and `make test`.
3. `make check-codex-compat` should continue to fail until semantic review and fixes are complete; the baseline must not be refreshed prematurely.

### Phase 5: Real Codex/API Testing

1. Use `deepseek-v4-flash` by default; use controlled model catalog override for Lite/code-mode to ensure that the target wire shape is actually triggered instead of just testing the unknown-model fallback.
2. Merge execution text/tool/Lite, patch error recovery, deferred plugin/MCP, web search, phase, reasoning, compact/resume/fork and concurrent window scenarios, and backfill the results in the above table one by one.
3. Do not trigger sensitive real prompts for `bio_policy`; use fixtures instead. Exercise live error gates with safe invalid parameters or model paths and record why they were substituted.

### Phase 6: Closing and version upgrade

After all fixed, automated and triggered real tests pass, update the contract baseline, this report, the current compatible baseline and version, and upgrade the Codex-Rosetta package version from `0.142.5` to `0.144.0.r0`.
